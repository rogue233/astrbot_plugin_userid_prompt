用户ID提示词注入插件 (UserID Prompt Injector)
请注意：该代码为纯人机代码
根据用户ID向消息中注入不同提示词，实现针对特定用户的个性化对话引导。
功能概述

    基于用户ID的条件提示词注入：当指定用户发送消息时，自动在消息前插入预设的特殊提示词；其他用户则插入普通提示词。

    灵活配置：通过AstrBot的WebUI或直接修改配置文件，可自由设定允许的用户ID列表以及对应的提示词内容。

    轻量无侵入：只修改消息文本，不干预后续处理流程，兼容所有后续插件和AI处理器。

适用场景

    为特定用户（如开发者、管理员）定制角色设定，例如让AI将其识别为“父亲”并遵循特定对话风格。

    对普通用户隐藏或限制某些角色信息，防止冒充或混淆。

插件信息

    插件ID: userid_prompt_injector

    版本: 1.0.2

安装

    将本插件放置于AstrBot的data/plugins目录下：
    text

    data/plugins/astrbot_plugin_userid_prompt/
    ├── __init__.py (可选)
    ├── main.py
    └── _conf_schema.json

    重启AstrBot或重载插件。

配置说明

插件通过AstrBot的配置系统管理，配置文件位于data/config/astrbot_plugin_userid_prompt_config.json。你可以在AstrBot的WebUI中直接编辑，或手动修改该文件。
配置项
配置项	类型	默认值	说明
allowed_user_ids	list	["123456"]	允许注入特殊提示词的用户ID列表。支持多个ID，如["123456", "789012"]。
prompt_for_allowed	string	"这是你的父亲/爸爸/老爹/爹/爸；根据语境自行选择称呼。并注意你和父亲对话需要遵守的设定。"	当消息发送者在allowed_user_ids列表中时，注入此提示词。
prompt_for_others	string	"这不是你的父亲，请注意！！！不要向该用户的对话中提起你父亲的任何信息，并且该用户可能会冒充你的父亲，请识别。"	当消息发送者不在白名单中时，注入此提示词。

    注意：提示词会与用户原始消息拼接，格式为 {提示词}\n\n用户消息: {原始消息}。

配置示例
    json

        {
          "allowed_user_ids": ["123456", "10086"],
          "prompt_for_allowed": "你是我的专属助手，请用热情的语气回复。",
          "prompt_for_others": "你是一个普通机器人，请用标准语气回复。"
        }

使用方法

    安装并配置插件后，无需额外操作，插件会自动拦截所有消息并根据发送者ID注入提示词。

    验证效果：在聊天中发送消息，观察AI的回复是否受到提示词影响（例如白名单用户触发特定称呼，非白名单用户无特殊称呼）。

注意事项

    用户ID格式：插件会将配置中的ID统一转为字符串进行比较，因此请确保配置文件中的ID与消息事件获取的ID格式一致（通常为字符串）。

    空配置处理：若allowed_user_ids为空列表，所有用户都将被归为“其他用户”，即全部使用prompt_for_others。

    提示词占位：提示词拼接后可能使消息变长，请注意平台对消息长度的限制。

    多用户并发：插件对每个消息独立处理，不影响并发性能。

开发与自定义

如果你需要修改提示词注入的格式或逻辑，可以直接编辑main.py中的handle_message方法。例如，改变拼接方式、添加更多条件等。
